Screen Navigation Map: retail-shop

This report documents the page-to-page transitions and navigation structure of the Multi-Shop Retail Management System, grounded in the React Router configuration and module architecture.

Application Navigation Overview

The application follows a Hub-and-Spoke navigation model with two distinct layouts:

  • Console Layout (/console/*): Admin interface with persistent sidebar, housing all management modules
  • POS Layout (/pos): Full-screen point-of-sale terminal, no sidebar, standalone peer to console

The sidebar provides role-based access to 12 retail modules. Individual modules follow standard CRUD (List → Add/View/Edit) patterns, while lifecycle modules (PO, GRN, Returns, Transfers) add state machine transitions on their View pages.

Authentication Flow

Authentication is managed via JWT tokens with username/password login.

  • Login Page: /auth/login src/client/pages/auth/Login.tsx
  • Form Fields: username, password
  • Success Outcome: JWT cookie set, redirect to /console/dashboard
  • Failure Outcome: Error message displayed on same page
  • Logout: POST to /api/auth/logout, redirect to /auth/login

    Start((Start)) --> Login["/auth/login"]
    Login -- "POST (Valid)" --> Dashboard["/console/dashboard"]
    Login -- "POST (Invalid)" --> LoginErr["/auth/login + error"]

    Register -- "POST (Success)" --> RegTenant["/auth/register-tenant"]
    RegTenant -- "POST (Success)" --> Login

    Forgot -- "POST" --> Reset["/auth/reset-password"]
    Reset -- "POST (Success)" --> Login

TL;DR: The retail-shop application is a PERN-stack (PostgreSQL, Express, React, Node.js) multi-tenant retail management system. Navigation is driven by a role-protected sidebar leading to 12 retail modules. Master data modules follow standard CRUD flows. Lifecycle modules (PO, GRN, Returns, Transfers) add state machine transitions on their View pages. The POS operates as a standalone full-screen terminal with its own sub-flows for checkout, shifts, hold/recall, and offline sync. All forms redirect back to their module's list page upon success.
